🚀 Bitcoin Bastion News Bot

Bitcoin Bastion News Bot — это многофункциональный Telegram-бот для криптоаналитики. Он в реальном времени отслеживает новости из мира криптовалют, анализирует их тональность с помощью AI и определяет, какие из них спровоцировали значимое движение цены. Такие «триггерные» новости автоматически публикуются в канал. Бот также предоставляет пользователям широкий спектр рыночных метрик, умеет работать в группах и имеет удобную панель администратора.

Бот построен на базе бесплатного API free-crypto-news, который агрегирует новости из 200+ источников (включая международные) и предоставляет готовые AI-эндпоинты (тональность, суммаризация, ответы на вопросы, извлечение сущностей и др.).
Для получения исторических цен используется CoinGecko.

✨ Возможности

📡 Реалтайм‑новости и AI‑анализ

· Подключение к SSE‑стриму API для получения новостей о Bitcoin и других криптовалютах в реальном времени.
· Для каждой новости запрашивается тональность (/api/ai/sentiment).
· Если тональность совпадает с направлением движения цены в течение заданного времени (настраивается), новость помечается как «спровоцировавшая движение» и публикуется в канал.

📊 Рыночные метрики (личные чаты)

· /btc – цена BTC, индекс страха и жадности, доминация BTC и ETH.
· /whales – последние китовые транзакции.
· /liquidations – последние ликвидации.
· /funding – ставки фандинга (8h).
· /gainers / /losers – топ движений рынка.
· /coin – детальная информация о монете (как на CoinGecko).
· /heatmap – тепловая карта рынка.
· /options – данные по опционам.
· /orderbook – стакан ордеров.
· /feargreed – индекс страха и жадности.
· /dominance – доминация BTC и ETH.

📰 Работа с новостями

· /latest – последние 5 новостей.
· /historical <тикер> – поиск в архиве новостей (с 2017 года).
· /international <код_языка> – международные новости с автоматическим переводом на английский (поддерживаются 18 языков).

🧠 AI‑инструменты

· /ask <вопрос> – задать вопрос AI по криптоновостям.
· /summarize <url> – получить краткое содержание новости.
· /factcheck <текст> – проверить факты в утверждении.
· /entities <текст> – извлечь ключевые сущности (компании, персоны, проекты).

👥 Групповой чат

· Публикация всех свежих новостей в указанную группу с настраиваемым интервалом (по умолчанию раз в час).
· Команда /latest_group для ручного получения последних новостей.

🔔 Подписки (личные уведомления)

Пользователи могут подписаться на:

· 🐋 Китовые транзакции
· 💥 Ликвидации
· ⚡ Триггерные новости (совпавшие с рынком)
· 📚 Архивные новости по выбранным монетам
· 🌍 Международные новости
· 🤖 AI‑оповещения

Уведомления приходят в личные сообщения бота.

🛠 Панель администратора

Доступна только пользователям из списка ADMIN_IDS или помеченным как администраторы в БД.

· Статистика: количество пользователей, новостей, триггерных событий, точность за 7 дней, разбивка по монетам.
· Настройка триггера: изменение порога изменения цены и временного интервала.
· Рассылка: отправка сообщений всем пользователям или публикация в канал вручную.
· Бэктестинг: анализ точности предсказаний за последние 30 дней с графиком изменения цены BTC.

🌐 Мультиязычность

· Поддерживаются русский и английский языки.
· Язык выбирается командой /language или автоматически определяется при первом запуске (на основе переменной DEFAULT_LANGUAGE в .env).

⚡ Кэширование

· Redis используется для кэширования рыночных данных и результатов API, что снижает нагрузку и ускоряет ответы.

📦 Установка и запуск

Предварительные требования

· Установленный Python 3.9 или выше.
· Установленный и запущенный Redis (например, redis-server).
· PostgreSQL (локально или облачный сервис, например Supabase или Neon.tech).
· Telegram Bot Token (получить у @BotFather).
· Telegram канал (для публикаций), в который бот добавлен как администратор.
· Telegram группа (опционально), куда бот будет постить все новости.

Шаги по установке

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/Hegehub/Bitcoin-Bastion-News-bot.git
   cd Bitcoin-Bastion-News-bot
   ```
2. Создайте виртуальное окружение и активируйте его:
   ```bash
   python -m venv venv
   source venv/bin/activate      # Linux/Mac
   venv\Scripts\activate         # Windows
   ```
3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
4. Создайте базу данных PostgreSQL и пользователя. Например, через psql:
   ```sql
   CREATE USER ultra_bot_user WITH PASSWORD 'your_password';
   CREATE DATABASE ultra_bot_db OWNER ultra_bot_user;
   ```
   Или используйте облачный сервис и скопируйте строку подключения.
5. Создайте файл .env в корне проекта и заполните его (см. раздел «Конфигурация» ниже).
6. Убедитесь, что Redis запущен:
   ```bash
   redis-server
   ```
7. Запустите бота:
   ```bash
   python bot.py
   ```

Бот готов к работе! Первый запуск автоматически создаст все необходимые таблицы в PostgreSQL.

⚙️ Конфигурация (файл .env)

Создайте файл .env в корне проекта и укажите следующие переменные:

```ini
BOT_TOKEN=ваш_токен_бота
CHANNEL_ID=@имя_канала_или_id
GROUP_CHAT_ID=-1001234567890        # ID группы (обязательно с минусом)
ADMIN_IDS=123456789,987654321       # ID администраторов через запятую
REDIS_URL=redis://localhost:6379/0

# PostgreSQL (пример для локального или облачного сервера)
DATABASE_URL=postgresql+asyncpg://user:password@localhost:5432/ultra_bot_db

# Настройки определения «триггерных» новостей
TRIGGER_PRICE_CHANGE_PERCENT=2.0    # минимальное изменение цены в %
TRIGGER_TIMEFRAME_MINUTES=30        # временной интервал для проверки (в минутах)

# Язык по умолчанию (en или ru)
DEFAULT_LANGUAGE=ru

# API ключ для CoinGecko (опционально, но с ним выше лимиты)
COINGECKO_API_KEY=ваш_ключ
```

Примечания:

· CHANNEL_ID может быть либо username канала (например, @mycryptochannel), либо его числовой ID (начинается с -100).
· GROUP_CHAT_ID также должен начинаться с -100.
· ADMIN_IDS – список числовых ID пользователей Telegram, которые получат доступ к админ‑командам.
· COINGECKO_API_KEY – необязателен, но с ним выше лимиты запросов.

🧠 Как работает определение «триггерных» новостей

1. Бот получает новость из SSE‑потока или периодически запрашивает последние новости через API.
2. Сохраняет новость в БД вместе с временем публикации.
3. Запрашивает тональность новости через эндпоинт /api/ai/sentiment.
4. Через заданный интервал (TRIGGER_TIMEFRAME_MINUTES) проверяет изменение цены указанной монеты (через CoinGecko). Если изменение превышает TRIGGER_PRICE_CHANGE_PERCENT и совпадает по направлению с тональностью, новость считается триггерной.
5. Триггерные новости публикуются в канал и отправляются подписчикам.

Важно: Для анализа цен используется CoinGecko API, которое имеет ограничение на точность исторических данных – для давних дат доступна только цена на начало дня. Это означает, что для timeframe менее 24 часов точность может быть снижена. В будущем планируется интеграция с более точными источниками.
