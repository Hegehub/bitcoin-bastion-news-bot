# Crypto Analytics Bot — Telegram-бот для криптоаналитики на базе Bitcoin-Bastion-News-API

**Crypto Analytics Bot** — это многофункциональный Telegram-бот, который в реальном времени анализирует криптоновости, сопоставляет их с движением рынка и предоставляет пользователям широкий спектр рыночных данных. Бот построен на базе бесплатного API [Bitcoin-Bastion-News-API](https://github.com/Hegehub/Bitcoin-Bastion-News-API) (также известного как Free Crypto News API) и использует его возможности для получения новостей, анализа тональности, суммаризации, а также рыночных метрик (страх и жадность, доминация BTC, ликвидации, китовые транзакции и др.).

## Возможности

### Основной цикл (работа в реальном времени)
- Подключение к `/api/stream` для получения новостей о Bitcoin в режиме реального времени (Server-Sent Events).
- Для каждой новости автоматически запрашивается тональность (`/api/ai/sentiment`).
- Новость сохраняется в базу данных вместе с текущей ценой BTC.
- Запускаются таймеры для обновления цены через 1, 6 и 24 часа.
- По истечении 1 часа вычисляется изменение цены, и новость помечается как «совпавшая», если её тональность совпала с направлением движения рынка.
- Совпавшие новости публикуются в указанный Telegram-канал (раз в 30 минут).

### Команды для пользователей (личные чаты и группы)
- `/latest` — последние 5 новостей о Bitcoin.
- `/btc` — текущая цена BTC, индекс страха и жадности, доминация Bitcoin.
- `/analyze <url>` — анализ тональности и краткое содержание новости по ссылке.
- `/search <запрос>` — полнотекстовый поиск по новостям.
- `/feargreed` — индекс страха и жадности.
- `/dominance` — доминация Bitcoin.
- `/liquidations` — последние ликвидации.
- `/whales` — последние китовые транзакции.
- `/subscribe` — управление подписками (уведомления о китах, ликвидациях, новостях с высокой тональностью).

### Административные функции
- `/admin` — панель администратора (доступна только пользователям из списка `ADMIN_IDS` или с флагом `is_admin` в БД).
- Статистика бота (количество пользователей, новостей, совпадений).
- Отправка сообщений в канал напрямую через бота.

### Веб-интерфейс
- Простой веб-дашборд на FastAPI для мониторинга статистики (общее количество новостей, совпадений, опубликованных записей).
- Доступен по адресу `http://localhost:8000`.

### Уведомления для подписчиков
- Пользователи могут подписаться на:
  - 🐋 Китовые транзакции.
  - 💥 Ликвидации.
  - 📰 Новости с высокой тональностью (совпавшие с рынком).
- Уведомления приходят в личные сообщения бота.

### Кэширование
- Redis используется для кэширования текущей цены BTC и индекса страха и жадности, чтобы снизить нагрузку на внешние API.

## Технологии
- **Python 3.9+**
- **aiogram** — асинхронный фреймворк для Telegram Bot API.
- **aiohttp** / **aiohttp-sse-client** — для работы с API и SSE-стримом.
- **SQLAlchemy 2.0 + aiosqlite** — асинхронная ORM и база данных.
- **Redis** — кэширование.
- **APScheduler** — планировщик задач (публикация в канал, проверка китов/ликвидаций).
- **FastAPI + Jinja2** — веб-интерфейс.
- **python-dotenv** — управление переменными окружения.

## Установка и настройка

### Предварительные требования
- Установленный Python 3.9 или выше.
- Установленный и запущенный Redis (например, `redis-server`).
- Telegram Bot Token (получить у [@BotFather](https://t.me/BotFather)).
- Telegram канал (для публикаций), в который бот добавлен как администратор.

### Шаги по установке

1. **Клонируйте репозиторий**:
   ```bash
   git clone https://github.com/yourusername/crypto-analytics-bot.git
   cd crypto-analytics-bot
```

1. Создайте виртуальное окружение и активируйте его:
   ```bash
   python -m venv venv
   source venv/bin/activate  # Linux/Mac
   venv\Scripts\activate     # Windows
   ```
2. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
3. Создайте файл .env в корне проекта и заполните его:
   ```env
   BOT_TOKEN=ваш_токен_бота
   CHANNEL_ID=@имя_канала_или_id
   ADMIN_IDS=123456789,987654321  # ID администраторов через запятую
   REDIS_URL=redis://localhost:6379/0
   ```
4. Убедитесь, что Redis запущен:
   ```bash
   redis-server
   ```
5. Запустите бота:
   ```bash
   python bot.py
   ```
6. Запустите веб-интерфейс (в отдельном терминале):
   ```bash
   uvicorn web:app --reload --port 8000
   ```
   После этого веб-интерфейс будет доступен по адресу http://localhost:8000.
